# market-basket-analysis

Анализ потребительской корзины на синтетическом датасете розничных транзакций.

Находит, какие товары покупают вместе, через **support, confidence и lift** — на SQL (self-join по гранулярности корзины) поверх **PostgreSQL**, с визуализацией. Генератор подмешивает реальные товарные бандлы в случайный шум, поэтому анализ должен *восстановить* сигнал. Без LLM и без ML-библиотек — только SQL и математика ассоциативных правил.

## Цель проекта

Ритейлеру нужно знать, **какие товары покупают вместе**, чтобы строить:

- рекомендации «с этим товаром покупают» и кросс-сейл;
- выкладку на полке / странице комплементарных товаров;
- дизайн промо (скидка на один товар тянет за собой комплемент).

Проект берёт сырые данные о корзинах, выявляет сильнейшие товарные ассоциации, ранжирует их и превращает в направленные правила кросс-сейла.

## Данные

Датасет **синтетический и полностью воспроизводимый** (фиксированный seed), генерируется [`src/generate_data.py`](src/generate_data.py) и сразу грузится в PostgreSQL.

| Таблица | Гранулярность | Поля |
|---|---|---|
| `products` | товар | `product_id`, `product_name`, `category` |
| `transactions` | корзина | `transaction_id`, `customer_id`, `transaction_date` |
| `transaction_items` | корзина × товар | `transaction_id`, `product_id` |

Генератор закладывает **9 реальных бандлов** (например, Coffee+Milk+Sugar, Pasta+Tomato Sauce+Parmesan, Chips+Soda+Salsa, Shampoo+Conditioner, классический Diapers+Beer) поверх независимых случайных покупок. Всё, что *не* бандл, — шум, поэтому рабочий анализ должен отделить настоящие комплементы от совпадений. По умолчанию: **12 000 корзин / 35 товаров / ~57 тыс. строк корзин**, средняя корзина ≈ 4.7 позиции.

## Метод: support, confidence, lift

Для неупорядоченной пары товаров **{A, B}** по **N** корзинам:

| Метрика | Формула | Как читать |
|---|---|---|
| **support(A,B)** | корзин с обоими ÷ N | как *часто* встречается пара |
| **confidence(A→B)** | корзин с обоими ÷ корзин с A | если в корзине A, насколько вероятен B |
| **lift(A,B)** | support(A,B) ÷ (support(A)·support(B)) | во сколько раз *чаще случайного* они вместе |

- **lift > 1** → комплементы (вместе чаще случайного)
- **lift ≈ 1** → независимы
- **lift < 1** → субституты / редко вместе

Confidence **направленный** (A→B ≠ B→A) — именно это делает рекомендации кросс-сейла применимыми. Фильтр по минимальному support убирает редкие, статистически шумные пары.

Ядро анализа — один self-join по гранулярности корзины, каждая неупорядоченная пара считается один раз:

```sql
from transaction_items a
join transaction_items b
    on a.transaction_id = b.transaction_id
   and a.product_id < b.product_id
```

Прочие приёмы: `COUNT(DISTINCT …)` для support, формула lift из счётчиков товаров и пар, фильтр по минимальному support, `UNION ALL` для разворота пар в направленные правила и `ROW_NUMBER() OVER (…)` для выбора лучшего консеквента на товар. См. [`sql/`](sql/).

## Как запустить

Нужны Docker (для PostgreSQL) и Python 3.

```bash
docker compose up -d                 # поднимает PostgreSQL 16 на host-порту 5443
cp .env.example .env                 # опционально; дефолты уже совпадают с compose

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python src/generate_data.py          # создаёт таблицы и грузит синтетические корзины
python src/analyze.py                # пишет графики + reports/basket_report.md
```

Посмотреть данные напрямую: `docker compose exec db psql -U analytics -d basket`.

## Выводы и рекомендации

1. **Метод восстанавливает настоящие комплементы.** Топ пар по lift — ровно заложенные бандлы. Шум остаётся на lift ≈ 1. Это подтверждает, что support/confidence/lift отделяют сигнал от совпадения.
2. **Ранжировать по lift, масштаб — по support.** Lift находит *связь*, support говорит, как *часто* это окупается. Лучшие цели сильны по обоим (например, Coffee+Sugar: lift 3.2× при support 11.8%).
3. **Кросс-сейл и выкладка.** Использовать таблицу правил на товар для виджетов «с этим покупают» и совместной выкладки, чтобы поднять attach rate.
4. **Дизайн промо.** Скидка на один товар сильной пары тянет комплемент по полной марже — нельзя скидывать оба сразу.
