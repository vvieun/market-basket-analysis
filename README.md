# market-basket-analysis

Market basket analysis on a synthetic retail transactions dataset.

Finds which products are bought together using **support, confidence and lift** — computed in pure SQL (self-join on the basket grain) on **PostgreSQL**, then visualised. The generator injects real product bundles into random noise, so the analysis has to *recover* the signal.

## Project goal

Retailers want to know **what products customers buy together**, to drive:

- "frequently bought together" recommendations and cross-sell;
- shelf / page placement of complementary products;
- promo design (discount one item, pull through its complement).

This project takes raw basket data and surfaces the strongest product associations, ranks them, and turns them into directional cross-sell rules.

## Data

The dataset is **synthetic and fully reproducible** (fixed random seed), generated by [`src/generate_data.py`](src/generate_data.py) and loaded straight into PostgreSQL.

| Table | Grain | Fields |
|---|---|---|
| `products` | product | `product_id`, `product_name`, `category` |
| `transactions` | basket | `transaction_id`, `customer_id`, `transaction_date` |
| `transaction_items` | basket × item | `transaction_id`, `product_id` |

The generator plants **9 real bundles** (e.g. Coffee+Milk+Sugar, Pasta+Tomato Sauce+Parmesan, Chips+Soda+Salsa, Shampoo+Conditioner, the classic Diapers+Beer) on top of independent random purchases. Everything that is *not* a bundle is noise — so a working analysis must separate genuine complements from coincidence. Default size: **12,000 baskets / 35 products / ~57k basket lines**, avg basket ≈ 4.7 items.

## Method: support, confidence, lift

For an unordered pair of items **{A, B}** over **N** baskets:

| Metric | Formula | Reads as |
|---|---|---|
| **support(A,B)** | baskets with both ÷ N | how *often* the pair occurs |
| **confidence(A→B)** | baskets with both ÷ baskets with A | if A is in the basket, how likely is B |
| **lift(A,B)** | support(A,B) ÷ (support(A)·support(B)) | how much *more than chance* they co-occur |

- **lift > 1** → complements (bought together more than chance)
- **lift ≈ 1** → independent
- **lift < 1** → substitutes / rarely together

Confidence is **directional** (A→B ≠ B→A), which is what makes cross-sell recommendations actionable. A minimum-support filter removes rare, statistically noisy pairs.

The core of the analysis is one self-join on the basket grain — each unordered pair counted once:

```sql
from transaction_items a
join transaction_items b
    on a.transaction_id = b.transaction_id
   and a.product_id < b.product_id
```

Other techniques: `COUNT(DISTINCT …)` for support, a lift formula assembled from item and pair counts, a min-support filter, `UNION ALL` to unfold pairs into directional rules, and `ROW_NUMBER() OVER (…)` to keep the single best consequent per product. See [`sql/`](sql/).

## How to run

Requires Docker (for PostgreSQL) and Python 3.

```bash
docker compose up -d                 # starts PostgreSQL 16 on host port 5443
cp .env.example .env                 # optional; defaults already match compose

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python src/generate_data.py          # creates tables and loads synthetic baskets
python src/analyze.py                # writes figures + reports/basket_report.md
```

To inspect the data directly: `docker compose exec db psql -U analytics -d basket`.

## Conclusions & recommendations

1. **The method recovers genuine complements.** The top-lift pairs are exactly the planted bundles. Noise stays at lift ≈ 1. This validates support/confidence/lift as a way to separate signal from coincidence.
2. **Rank on lift, size on support.** Lift finds the *relationship*, support tells you how *often* it pays off. The best targets score on both (e.g. Coffee+Sugar: 3.2× lift at 11.8% support).
3. **Cross-sell & placement.** Drive "frequently bought together" widgets and co-locate shelf placement from the per-product rule table to raise attach rate.
4. **Promo design.** Discount one item in a high-lift pair to pull through its complement at full margin — never discount both at once.
